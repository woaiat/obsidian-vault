## 1. 结构化数据：Structs & Uniform Initialization
在 Modern C++ 中，`struct` 不仅仅是数据的堆叠，更是封装的起点。

### 1.1 统一初始化 (Uniform Initialization)
自 C++11 起，推荐使用大括号 `{}` 进行初始化，它可以防止 **Narrowing Conversion**（精度丢失的隐式转换）。
```cpp
struct Student {
    std::string name;
    double gpa;
    int age;
};

// 推荐做法
Student s1{"Alice", 3.9, 19}; 
// 如果写 int age = {3.5}; 编译器会直接报错，而不会隐式转换成 3，这在安全编程中很重要。
```

---

## 2. STL 宏观概览 (STL Overview)
STL (Standard Template Library) 的核心哲学是 **容器（Data）与算法（Logic）的分离**，而连接它们的桥梁是 **迭代器（Iterators）**。

*   **容器 (Containers)**：存储数据。
*   **算法 (Algorithms)**：处理数据（如 `std::sort`, `std::find`）。
*   **迭代器 (Iterators)**：像指针一样的对象，用来遍历容器。

---

## 3. 序列容器 (Sequence Containers)
序列容器中的元素按照逻辑线性排列，你掌握的重点应放在 **内存布局** 和 **复杂度分析** 上。

### 3.1 `std::vector<T>`：动态连续数组 速度最快
**底层原理**：一块连续的内存。当容量不足时，申请一块约 $1.5$ 到 $2$ 倍大的新内存，将旧元素移动（Move）过去。

#### 📝 常用操作（增删查改）
| 操作类型 | 函数 | 时间复杂度 | 备注 |
| :--- | :--- | :--- | :--- |
| **增加** | `push_back(val)` | $O(1)$ (均摊) | 在末尾添加 |
| | `insert(it, val)` | $O(n)$ | 需要移动插入点后的所有元素 |
| | `emplace_back(args)` | $O(1)$ | **进阶**：直接在原地构造，减少拷贝 |
| **删除** | `pop_back()` | $O(1)$ | 弹出末尾 |
| | `erase(it)` | $O(n)$ | 删除后需移动后续元素 |
| | `clear()` | $O(n)$ | 清空所有元素 |
| **查询/访问** | `v[i]` | $O(1)$ | 不进行边界检查（快但危险） |
| | `v.at(i)` | $O(1)$ | **安全**：越界会抛出异常 |
| | `v.front() / v.back()` | $O(1)$ | 访问首尾 |
| **容量** | `v.size()` | $O(1)$ | 当前元素个数 |
| | `v.capacity()` | $O(1)$ | 当前底层分配的总空间 |

### 3.2 `std::deque<T>`：双端队列 (Double-Ended Queue)
**底层原理**：由多个分段连续的“数据块”组成的映射结构。它结合了 `vector` 和 `list` 的部分优点。

#### 📝 常用操作
*   **比 vector 多出的核心操作**：
    *   `push_front(val)`：$O(1)$ 时间内在头部插入。
    *   `pop_front()`：$O(1)$ 时间内弹出头部。
*   **注意点**：虽然支持 `[]` 随机访问，但由于内存不完全连续，其速度略慢于 `vector`。

### 3.2. `std::list<T>` :双向链表:不支持随机访问 速度最慢
- 其 `insert()` 从指向元素前插入 `remove()` 可指定元素删除

## 4. 容器适配器 (Container Adaptors)
适配器并不直接管理内存，而是“套”在序列容器外面，限制其功能以实现特定的逻辑。
- stack和queue不适配迭代器 不能用增强for循环

| 适配器                       | 底层默认容器   | 逻辑特性        | 常用指令                                   |
| :------------------------ | :------- | :---------- | :------------------------------------- |
| **`std::stack`**          | `deque`  | LIFO (后进先出) | `push()`, `pop()`, `top()`             |
| **`std::queue`**          | `deque`  | FIFO (先进先出) | `push()`, `pop()`, `front()`, `back()` |
| **`std::priority_queue`** | `vector` | 自动排序（默认大顶堆） | `push()`, `pop()`, `top()`             |
|                           |          |             |                                        |
|                           |          |             |                                        |
- 容器的底层默认容器可以修改 表面上对程序无影响 但是 **底层对代码运行效率有影响**
 `stack<int,vector<int>> st` `queue<int,list<int> que` 
- == **stack三个都可以选 queue只能选deque和list priority_queue只能选vector和deque** ==

## 💡 深度思考：安全研究员视角 (C9 级进阶)

### 1. `size()` vs `capacity()` 的安全隐患
在处理网络数据包时，如果你预先知道数据大小，务必使用 `v.reserve(n)`。
*   **原因**：避免频繁扩容导致的性能损耗，以及防止多次内存分配产生的碎片。

### 2. `at(i)` 是你最好的朋友
作为未来的安全研究员，你应该形成条件反射：**除非在极端追求性能的内层循环，否则永远优先使用 `.at()` 而不是 `[]`**。
*   **漏洞溯源**：无数的缓冲区溢出（Buffer Overflow）漏洞就是因为开发者信任了索引值而没有做边界检查。

# 3.2 ：Associative Containers and Iterators

## ==Associative Containers== 


## 一、核心定义 & 核心特征

### 定义

关联容器是 **C++ STL 标准库的四大容器类别之一**（另外三个：序列容器、容器适配器、无序容器），本质是**以「键-值(key-value)」为核心的存储结构**，容器的组织、查找、删除都围绕**键(key)** 进行，而非序列容器的「插入位置」。

### 所有关联容器的通用核心特点（重中之重）

1. **核心能力：按「键」快速查找** —— 这是关联容器存在的意义，查找效率远高于序列容器（vector/list 等）；
    
2. **键的不可修改性**：容器中存储的元素的「键」是 **const 只读属性**，**绝对不能被修改**！如果必须改变某个键，只能「先删除旧键的元素，再插入新键的元素」。
    
    1. ❓ 为什么键不能改？因为关联容器的底层结构是基于「键的排序/哈希值」构建的，修改键会直接破坏容器的底层结构，导致容器失效；
        
3. **元素是有序存储的**（仅 **有序关联容器**，这也是它的核心标识）；
    
4. **迭代器类型**：仅支持「双向迭代器」，**不支持随机访问迭代器** —— 不能用 `it + 5` / `it - 3` 这种随机访问操作，只能用 `++it` / `--it` 进行前后遍历。
    

---

## 二、关联容器的核心分类

C++ 中的关联容器，**标准划分为两大体系**，二者功能互补，也是面试高频考点：

### 分类依据

底层实现的 **数据结构不同**，导致「有序性、性能、适用场景」完全不同。

1. **有序关联容器 (Sorted Associative Containers)** —— C++98 原生支持，STL 核心基础容器
    
2. **无序关联容器 (Unordered Associative Containers)** —— C++11 新增，也叫「哈希容器」，是对有序关联容器的性能补充
    

---

## 三、第一类：有序关联容器（核心！必学必用）

### 核心共性（全部满足，无例外）

1. 底层统一使用 **红黑树 (Red-Black Tree)** —— 一种「自平衡的二叉搜索树」；
    
2. 容器内的元素，**永远按照「键(key)」的大小进行升序排序**（默认排序规则：`std::less<Key>`，可自定义排序规则）；
    
3. 所有操作（插入/删除/查找）的 **时间复杂度均为 O(log n)** —— 红黑树的平衡特性保证了稳定的对数级效率，n 是容器元素个数；
    
4. 键的唯一性：分为「唯一键」和「可重复键」两类子容器。

# 📑 关联容器成员函数全指南 (C++ Modern)

## 1. 核心成员函数对照表

下表涵盖了 `std::set`, `std::map`, `std::unordered_set`, `std::unordered_map` 共同且最常用的函数。

| 功能分类 | 函数名 | 有序容器复杂度 | 无序容器复杂度 | 备注 |
| :--- | :--- | :--- | :--- | :--- |
| **插入** | `insert(val)` | $O(\log n)$ | 平均 $O(1)$, 最坏 $O(n)$ | 插入元素（若已存在则无效） |
| | `emplace(args...)` | $O(\log n)$ | 平均 $O(1)$, 最坏 $O(n)$ | **推荐**：原地构造，效率最高 |
| **删除** | `erase(key)` | $O(\log n)$ | 平均 $O(1)$, 最坏 $O(n)$ | 按 Key 删除，返回删除个数 |
| | `clear()` | $O(n)$ | $O(n)$ | 清空容器 |
| **查找** | `find(key)` | $O(\log n)$ | 平均 $O(1)$, 最坏 $O(n)$ | 返回迭代器，找不到返回 `end()` |
| | `count(key)` | $O(\log n)$ | 平均 $O(1)$, 最坏 $O(n)$ | 返回个数（在非 Multi 容器中仅 0 或 1） |
| | `contains(key)` | $O(\log n)$ | 平均 $O(1)$, 最坏 $O(n)$ | **C++20 新增**，返回 bool，最简洁 |
| **容量** | `size()` | $O(1)$ | $O(1)$ | 当前元素个数 |
| | `empty()` | $O(1)$ | $O(1)$ | 检查是否为空 |

## 2. 详细功能深度拆解

### A. 访问元素 (仅限 Map 家族)
`std::map` 和 `std::unordered_map` 有独特的访问方式：

1.  **`operator[key]`**
    *   **语法**：`my_map["Alice"] = 100;`
    *   **特性**：如果 Key 不存在，它会**自动插入**一个默认值的元素。
    *   **危险点**：在 `const` 对象上无法使用，且容易在你不小心写错 Key 时产生冗余数据。

2.  **`at(key)`**
    *   **语法**：`int score = my_map.at("Alice");`
    *   **特性**：会进行边界检查。如果 Key 不存在，抛出 `std::out_of_range` 异常。
    *   **安全视角**：在处理不可信输入时，优先使用 `at()` 或 `find()`。

---

### B. 有序容器特有函数 (Set/Map)
由于有序容器（红黑树实现）是有序的，它们支持范围操作：

1.  **`lower_bound(key)`**：返回第一个 **$\ge$ key** 的迭代器。
2.  **`upper_bound(key)`**：返回第一个 **$>$ key** 的迭代器。
3.  **应用场景**：比如在一个安全日志系统中，快速查找某个时间段（从 10:00 到 12:00）的所有攻击记录。

---

### C. 无序容器特有函数 (Unordered家族)
由于无序容器是哈希表实现，你可以监控它的“健康状况”：

1.  **`load_factor()`**：返回当前的装载因子（$\frac{元素总数}{桶数}$）。如果这个值太高，哈希冲突会加剧。
2.  **`rehash(n)`**：强制扩容到 $n$ 个桶。
3.  **安全提示**：如果你发现 `load_factor` 很低但查询极慢，说明可能遭受了 **Hash DoS 攻击**。

---

## 3. Modern C++ 实战代码示例


```cpp
#include <iostream>
#include <unordered_map>
#include <map>
#include <string>

int main() {
    // 1. 初始化
    std::unordered_map<std::string, std::string> dns_cache;

    // 2. 插入 (使用 emplace 避免临时对象)
    dns_cache.emplace("google.com", "8.8.8.8");
    dns_cache.emplace("cloudflare.com", "1.1.1.1");

    // 3. C++17 结构化绑定遍历
    std::cout << "Current Cache:" << std::endl;
    for (const auto& [domain, ip] : dns_cache) {
        std::cout << domain << " -> " << ip << std::endl;
    }

    // 4. 安全查找 (C++20 风格)
    std::string target = "baidu.com";
    if (dns_cache.contains(target)) {
        std::cout << "Found: " << dns_cache.at(target) << std::endl;
    } else {
        std::cout << target << " not in cache. Inserting..." << std::endl;
        dns_cache[target] = "110.242.68.66";
    }

    // 5. 删除
    dns_cache.erase("google.com");

    return 0;
}
```

### 有序关联容器包含4个具体容器（头文件：`<set>` / `<map>`）

这4个容器两两成对，记忆成本极低：

#### 1. `std::set<Key>` —— 单值集合（键=值，唯一键）

- 存储规则：**只存「键」，键就是值，值就是键**（key = value）；
    
- 核心约束：**键是唯一的** → 容器中绝对不会存在重复的元素；
    
- 常用场景：去重、快速查找某个值是否存在、有序的无重复集合（如：存储唯一学号、不重复成绩）。
    
- 示例代码：
    

```C++
#include <set>
#include <iostream>
using namespace std;

int main() {
    set<int> s{3,1,4,1,5,9}; // 自动去重 + 自动排序
    for (int val : s) cout << val << " "; // 输出：1 3 4 5 9
    
    s.insert(2);  // 有序插入
    s.erase(4);   // 按键删除
    bool exist = s.count(5); // true，判断键是否存在
    return 0;
}
```

#### 2. `std::multiset<Key>` —— 多值集合（键=值，可重复键）

- 存储规则：和 `set` 完全一致，键=值；
    
- 核心约束：**键是可重复的** → 允许存在多个相同的键（元素）；
    
- 核心区别 vs `std::set`：仅差「是否允许重复键」，其余接口、用法、底层结构完全一致；
    
- 常用场景：有序存储且允许重复元素的场景（如：存储所有学生成绩，允许同分）；
    
- 核心接口：`count(Key)` —— 统计某个键出现的次数（set 中 count 只有 0/1）。
    

#### 3. `std::map<Key, Value>` —— 映射表（严格键值对，唯一键）

✅ 最常用的关联容器，没有之一

- 存储规则：存储 `std::pair<const Key, Value>` 结构体
    
    - `pair.first` → 键（Key），**const 只读，不可修改**；
        
    - `pair.second` → 值（Value），普通属性，**可以随意修改**；
        
- 核心约束：**键是唯一的** → 一个键只能对应唯一一个值（一对一映射）；
    
- 核心特性：支持 `[]` 运算符重载 → `map[key]`，直接通过键访问值；
    
- 常用场景：字典映射、查表、关联属性存储（如：学号→姓名、单词→翻译）。
    
- 示例代码：
    

```C++
#include <map>
#include <iostream>
using namespace std;

int main() {
    map<string, int> score; // 键：姓名，值：分数
    score["张三"] = 95;     // 插入键值对
    score["李四"] = 88;
    score["张三"] = 98;     // 键唯一，直接修改对应值
    
    // 遍历：pair.first 是 const string，pair.second 是 int
    for (auto& p : score) {
        cout << p.first << " : " << p.second << endl;
    }
    return 0;
}
```

#### 4. `std::multimap<Key, Value>` —— 多重映射表（键值对，可重复键）

- 存储规则：和 `map` 完全一致，存储 `std::pair<const Key, Value>`；
    
- 核心约束：**键是可重复的** → 一个键可以对应多个值（一对多映射）；
    
- 核心区别 vs `std::map`：
    
    - 仅差「是否允许重复键」；
        
    - **不支持** **`[]`** **运算符重载** → 一个键对应多个值，无法确定返回哪个值；
        
- 常用场景：一对多映射关系（如：姓名→多个电话号码、班级→多个学生）。
    

### 有序关联容器记忆口诀

set是单值，map是键值对；

前缀multi是重复键，无前缀是唯一键。

## 四、第二类：无序关联容器 (Unordered Associative Containers)

### 官方定位

C++11 新增容器类别，也叫「哈希关联容器」，是**有序关联容器的高性能版本**。

### 核心共性（全部满足，无例外）

1. 底层统一使用 **哈希表 (Hash Table)** —— 哈希桶 + 链表/红黑树的组合结构；
    
2. 容器内的元素是 **无序存储** 的 —— 存储顺序与键的大小无关，与键的哈希值有关，遍历顺序随机；
    
3. 核心性能优势：**平均时间复杂度为 O(1)** —— 插入/删除/查找效率为常数级，远快于有序容器的 O(log n)；
    
    1. ⚠️ 注意：最坏时间复杂度是 O(n)（哈希冲突严重时），实际开发中几乎不会出现；
        
4. 键的不可修改性：和有序关联容器完全一致，**键是 const 只读属性**；
    
5. 迭代器类型：仅支持「前向迭代器」，比有序容器的双向迭代器功能更弱，只能用 `++it` 向后遍历。
    

### 无序关联容器包含4个具体容器（头文件：`<unordered_set>` / `<unordered_map>`）

和有序关联容器 **一一对应**，功能完全相同，接口 99% 一致：

1. `std::unordered_set<Key>` → 对应 `std::set`：单值集合、唯一键、无序；
    
2. `std::unordered_multiset<Key>` → 对应 `std::multiset`：多值集合、可重复键、无序；
    
3. `std::unordered_map<Key, Value>` → 对应 `std::map`：键值对、唯一键、无序、支持 `[]` 运算符；
    
4. `std::unordered_multimap<Key, Value>` → 对应 `std::multimap`：键值对、可重复键、无序、无 `[]` 运算符。
    

### 记忆技巧

所有无序容器，都是在有序容器名字前加 `unordered_`，功能完全复用。

## 五、有序 vs 无序 关联容器 核心对比（面试必考）

|   |   |   |
|---|---|---|
|对比维度|有序关联容器 (set/map/multiset/multimap)|无序关联容器 (unordered_*)|
|底层数据结构|红黑树（自平衡二叉搜索树）|哈希表|
|元素存储顺序|按「键」升序排序，遍历有序|完全无序，遍历随机|
|核心时间复杂度|O(log n) 稳定效率|平均 O(1)，最坏 O(n)|
|迭代器支持|双向迭代器（++it / --it）|前向迭代器（仅 ++it）|
|查找效率|高效|**极致高效**（最优选择）|
|有序遍历需求|支持，天然有序|不支持，需手动排序|
|适用场景|需要**有序存储+查找**的场景|只需要**快速查找/插入**，不关心顺序的场景|

## 六、关联容器 vs 序列容器（vector/list/deque）核心区别

### 核心区别（唯一本质）

|   |   |   |   |   |
|---|---|---|---|---|
|容器类型|组织方式|访问方式|核心特性|查找时间复杂度|
|序列容器|按 **插入位置** 组织|靠「下标/迭代器位置」访问|强调「顺序」|O(n)|
|关联容器|按 **键(key)** 组织|靠「键的匹配」访问|强调「映射」|O(log n)/O(1)|

### 衍生区别

1. 序列容器无「键值」概念，元素是单纯的数值/对象；关联容器必须有「键」；
    
2. 序列容器元素可随意修改，无 const 限制；关联容器的「键」是 const 只读；
    
3. 序列容器支持随机访问（vector/deque），关联容器不支持。
    

---

## 七、核心使用场景选型建议（开发必看）

直接套用，覆盖 99% 场景：

1. 只需要「去重+查找」，不关心顺序 → 选 `unordered_set`（效率最高）；
    
2. 需要「去重+查找+有序遍历」→ 选 `set`；
    
3. 需要「重复元素+查找」，不关心顺序 → 选 `unordered_multiset`；
    
4. 需要「重复元素+查找+有序遍历」→ 选 `multiset`；
    
5. 需要「键值对映射+查找」，不关心顺序 → 选 `unordered_map`（开发首选）；
    
6. 需要「键值对映射+查找+有序遍历」→ 选 `map`；
    
7. 需要「一对多的键值映射」→ 无顺序用 `unordered_multimap`，有序用 `multimap`。
    

---

## 八、高频考点总结（面试必背）

1. 关联容器的核心是「键-值」存储，**键是 const 不可修改**；
    
2. 有序关联容器底层是红黑树，无序是哈希表；
    
3. 有序容器时间复杂度 O(log n)，无序容器平均 O(1)；
    
4. 关联容器不支持随机访问迭代器，只能双向/前向遍历；
    
5. `map` 支持 `[]` 运算符，`multimap` 不支持；
    
6. 无序容器是 C++11 新增，是有序容器的高性能替代品。
    

---

## 总结

1. 关联容器是 **按键访问** 的容器，核心能力是「快速查找」，键不可修改，不支持随机访问；
    
2. 分为**有序（红黑树，O(log n)，有序）** 和**无序（哈希表，O(1)，无序）** 两大体系，共 8 个具体容器，一一对应；
    
3. `set`/`map` 是「唯一键」，`multi` 前缀是「可重复键」，`set` 系列是单值，`map` 系列是键值对；
    
4. 开发中**优先选无序容器**（`unordered_*`），除非明确需要有序遍历，才选有序容器；
    
5. 关联容器和序列容器的本质区别是「按键访问」vs「按位置访问」。

## 一、迭代器 核心基础（必记）

### ✅ 迭代器的本质

C++的迭代器是一个**指针式对象**，封装了内存地址的访问逻辑，行为和指针高度相似；本质是「容器的通用访问/遍历工具」。

### ✅ 迭代器的核心价值

1. 为**所有STL容器**提供**统一的遍历/访问接口**，不管容器底层是数组/链表/红黑树，遍历语法一致；
    
2. 实现「容器」和「STL算法」的解耦，所有算法（sort/find/count）只接收迭代器参数，一套算法适配所有容器；
    
3. 弥补下标访问的短板：下标只支持连续内存容器，迭代器支持全部容器。
    

### ✅ 迭代器的核心声明语法

#### 1. 教学版 - 完整类型声明（必须理解，知道底层类型）

```C++
// 普通迭代器：可读、可修改指向的元素
容器类型::iterator 迭代器名;
// 常量迭代器：只读、不可修改，开发中90%场景用这个，更安全
容器类型::const_iterator 迭代器名;
// 反向迭代器：倒序遍历
容器类型::reverse_iterator 迭代器名;
// 常量反向迭代器：只读+倒序
容器类型::const_reverse_iterator 迭代器名;
```

#### 2. 开发版 - auto自动推导（无脑用，编译期推导，无性能损耗，功能完全一致）

```C++
auto 迭代器名 = 容器.begin(); // 自动推导为 容器::iterator
auto 迭代器名 = 容器.cbegin();// 自动推导为 容器::const_iterator
auto 迭代器名 = 容器.rbegin();// 自动推导为 容器::reverse_iterator
```

### ✅ 迭代器 核心通用操作符（所有迭代器都支持，必记）

迭代器的操作都是重载的运算符，行为和指针一致：

- `++it` ：前置自增，迭代器向前移动一位（**推荐写法，无开销**）
    
- `*it` ：解引用，获取迭代器指向的元素值（普通迭代器可读写，const迭代器只读）
    
- `it->`：成员访问，容器存对象/键值对时使用，等价于 `(*it).成员`
    
- `it1 == it2` / `it1 != it2` ：判断迭代器是否指向同一位置（遍历终止条件）
    

❗ 注意：`it++` 是后置自增，会生成临时对象，有额外拷贝开销，**所有场景一律用 ++it**

### ✅ 容器的2个核心迭代器入口（所有容器通用）

1. `begin()` ：返回迭代器，指向容器**第一个有效元素**
    
2. `end()` ：返回迭代器，指向容器**最后一个元素的下一个位置（尾后迭代器）**
    

⚠️ 重中之重：`end()` 是「哨兵」，不指向任何有效元素，**绝对不能解引用(*it)或调用it->**，唯一作用是判断遍历结束。

配套只读入口：`cbegin()`/`cend()` ，返回常量迭代器；倒序入口：`rbegin()`/`rend()`，返回反向迭代器。

## 二、迭代器的5大分类 & 容器对应关系（重中之重，无废话速记）

迭代器按「功能强弱」分级，**功能强的兼容功能弱的所有操作**，记牢容器对应的迭代器类型，开发不会出错

### ✅ 迭代器5类（功能从弱→强）

1. 输入迭代器：只读、单向向前 → 流迭代器(istream_iterator)
    
2. 输出迭代器：只写、单向向前 → 流迭代器(ostream_iterator)
    
3. 前向迭代器：可读可写、单向向前 → 对应容器：forward_list
    
4. 双向迭代器：可读可写、双向移动(支持++/--) → 对应容器：list、set、multiset、map、multimap
    
5. 随机访问迭代器【最强】：可读可写+双向+随机跳转(+/-/+=/-=) → 对应容器：vector、array、deque、原生数组
    

### ✅ 必记速查表（开发高频）

```Plain
✅ 随机访问迭代器：vector、array、deque、int arr[]
✅ 双向迭代器：list、set、map、multiset、multimap
✅ 前向迭代器：forward_list
```

#include <iostream>

### ✅ 场景1：普通迭代器 - 可读可写遍历（vector为例，连续内存容器）

作用：遍历容器 + 修改容器内的元素值

```C++
vector<int> vec = {10, 20, 30, 40};

// ===== 教学版本：完整迭代器类型 写法 =====
for (vector<int>::iterator it = vec.begin(); it != vec.end(); ++it) {
    *it *= 2;   // 解引用修改元素值
    cout << *it << " "; // 解引用访问元素值
}
cout << endl; // 输出：20 40 60 80

// ===== 开发版本：auto自动推导 写法 =====
for (auto it = vec.begin(); it != vec.end(); ++it) {
    *it *= 2;
    cout << *it << " ";
}
cout << endl; // 输出：40 80 120 160
```

### ✅ 场景2：常量迭代器 - 只读遍历（所有容器通用，开发最常用！）

作用：只读取元素，**禁止修改**，语法层面杜绝误修改，安全性拉满，90%的遍历场景都是这个写法

```C++
vector<int> vec = {1, 2, 3, 4};
list<int> li = {10, 20, 30};

// ===== 教学版本：const_iterator 完整写法 =====
// vector的const遍历
for (vector<int>::const_iterator cit = vec.cbegin(); cit != vec.cend(); ++cit) {
    cout << *cit << " "; // 只读，正常访问
    // *cit += 1; // 编译报错！常量迭代器禁止修改
}
cout << endl;

// list的const遍历（list无下标，只能用迭代器）
for (list<int>::const_iterator cit = li.cbegin(); cit != li.cend(); ++cit) {
    cout << *cit << " ";
}
cout << endl;

// ===== 开发版本：auto + cbegin/cend 简化写法 =====
// vector的const遍历
for (auto cit = vec.cbegin(); cit != vec.cend(); ++cit) {
    cout << *cit << " ";
}
cout << endl;

// list的const遍历
for (auto cit = li.cbegin(); cit != li.cend(); ++cit) {
    cout << *cit << " ";
}
cout << endl;
```

### ✅ 场景3：反向迭代器 - 倒序遍历（所有容器通用）

作用：从容器**最后一个元素 → 第一个元素**倒序遍历，反向迭代器的`++it`等价于原迭代器的`--it`

```C++
vector<int> vec = {1, 2, 3, 4};

// ===== 教学版本：reverse_iterator 完整写法 =====
for (vector<int>::reverse_iterator rit = vec.rbegin(); rit != vec.rend(); ++rit) {
    cout << *rit << " "; // 输出：4 3 2 1
}
cout << endl;

// 常量反向迭代器（只读倒序）
for (vector<int>::const_reverse_iterator crit = vec.crbegin(); crit != vec.crend(); ++crit) {
    cout << *crit << " ";
}
cout << endl;

// ===== 开发版本：auto + rbegin/rend 简化写法 =====
// 可写倒序
for (auto rit = vec.rbegin(); rit != vec.rend(); ++rit) {
    *rit += 10; // 修改元素
    cout << *rit << " "; // 输出：14 13 12 11
}
cout << endl;

// 只读倒序（推荐）
for (auto crit = vec.crbegin(); crit != vec.crend(); ++crit) {
    cout << *crit << " ";
}
cout << endl;
```

### ✅ 场景4：map/set 迭代器遍历（最能体现auto价值！重点）

核心说明：map/set这类关联容器**完全不支持下标访问**，迭代器是唯一遍历方式；

map的迭代器指向 `pair<const 键类型, 值类型>`，必须用 `it->first` 访问键，`it->second` 访问值；

set的迭代器指向的是元素本身，且是只读的（set元素不可修改）。

```C++
// map：键值对容器
map<string, int> stu = {{"张三", 20}, {"李四", 25}, {"王五", 22}};
// set：无重复元素容器
set<int> s = {5, 2, 8, 1};

// ===== 教学版本：map/set 完整迭代器写法（超长，开发绝不手写） =====
// map遍历
for (map<string, int>::iterator it = stu.begin(); it != stu.end(); ++it) {
    cout << "姓名：" << it->first << " 年龄：" << it->second << endl;
    it->second += 1; // 修改值（键是const，不可修改）
}
// set遍历（set的迭代器天然是const，不可修改元素）
for (set<int>::iterator it = s.begin(); it != s.end(); ++it) {
    cout << *it << " ";
}
cout << endl;

// ===== 开发版本：auto 简化写法（极致简洁，开发必用！） =====
// map遍历（可写）
for (auto it = stu.begin(); it != stu.end(); ++it) {
    cout << "姓名：" << it->first << " 年龄：" << it->second << endl;
}
// map只读遍历（推荐）
for (auto it = stu.cbegin(); it != stu.cend(); ++it) {
    cout << it->first << ":" << it->second << " ";
}
cout << endl;

// set遍历
for (auto it = s.cbegin(); it != s.cend(); ++it) {
    cout << *it << " ";
}
cout << endl;
```

### ✅ 场景5：指定区间遍历（迭代器灵活特性，下标做不到）

作用：只遍历容器的**某一段元素**，比如从第2个元素到倒数第2个元素，迭代器可自由指定遍历范围，下标也能实现但逻辑更复杂

```C++
vector<int> vec = {0, 1, 2, 3, 4, 5, 6};

// ===== 教学版本：完整类型 指定区间 =====
// 遍历：vec.begin()+2 到 vec.end()-2 （元素2,3,4）
for (vector<int>::iterator it = vec.begin()+2; it != vec.end()-2; ++it) {
    cout << *it << " ";
}
cout << endl;

// ===== 开发版本：auto 指定区间 =====
for (auto it = vec.begin()+2; it != vec.end()-2; ++it) {
    cout << *it << " ";
}
cout << endl;
```

## 四、C++11 终极简化 → 范围for循环（迭代器的语法糖，开发首选TOP1）

### ✅ 核心结论

范围for循环 `for(元素 : 容器)` 是**迭代器的语法糖**，编译器编译时会自动翻译成「迭代器遍历代码」，兼具：

- 下标遍历的**简洁语法**
    
- 迭代器的**全容器通用性**
    

适用场景：99%的「从头到尾完整遍历」场景，是开发中**最常用的遍历方式**，没有之一！

核心优势：不用写begin/end/++it，代码量最少，可读性最高。

### ✅ 范围for 三种写法（按优先级排序，全场景覆盖，均为auto版）

所有写法均为开发版本，**无教学版**，因为范围for本身就是简化语法，直接用即可

```C++
vector<int> vec = {1, 2, 3, 4};
map<string, int> stu = {{"张三",20}, {"李四",25}};

// ✔ 写法1：const auto &elem 【只读遍历，90%场景首选，推荐！】
// const：只读安全  &：引用避免拷贝，效率拉满，无任何副作用
for (const auto &x : vec) {
    cout << x << " ";
}
cout << endl;
// map的const范围for遍历
for (const auto &p : stu) {
    cout << p.first << ":" << p.second << " ";
}
cout << endl;

// ✔ 写法2：auto &elem 【可读可写遍历，修改元素，10%场景】
// 加引用即可修改原容器的元素，无拷贝开销
for (auto &x : vec) {
    x *= 10; // 修改元素值
    cout << x << " ";
}
cout << endl;

// ✔ 写法3：auto elem 【值拷贝遍历，极少用】
// 缺点：会拷贝每个元素，效率低；优点：语法最简单
// 适合元素是int/char这类小类型的简单场景
for (auto x : vec) {
    cout << x << " ";
}
cout << endl;
```

## 五、下标遍历 `[]` VS 迭代器遍历 核心区别（精华对比，彻底解惑）

你的核心疑问：为什么有简单的下标，还要用迭代器？这个对比表直接记死，无歧义！

### ✅ 核心对比（维度清晰，重点加粗）

|   |   |   |
|---|---|---|
|对比维度|下标遍历 `for(int i=0;i<size();i++)`|迭代器遍历（含auto/范围for）|
|适用容器|仅支持**连续内存容器**：vector/array/deque/原生数组|**所有STL容器通用**：vector/list/map/set全支持|
|底层原理|基于连续内存地址偏移：`*(首地址+i*元素大小)`|指针式对象直接指向元素，无需计算偏移|
|设计思想|强耦合：遍历代码和容器类型绑定，换容器要改遍历逻辑|完全解耦：遍历代码和容器类型无关，换容器无需改代码|
|对接算法|完全不支持：STL算法只接收迭代器参数|唯一对接方式：所有STL算法（sort/find/copy）都用迭代器|
|灵活性|差：只能从头到尾遍历，只能正向|极强：指定区间、反向、只读、读写自由切换|
|复杂容器|无法遍历map/set/list|唯一遍历方式，完美适配map键值对|
|效率|与迭代器**完全一致**（无性能差异）|与下标一致，auto无任何额外开销|

### ✅ 选用原则（无脑遵循，永不纠结）

1. 用下标遍历：仅当容器是**vector/array/deque** + 需要**用到索引i**（比如i%2、交换元素）；
    
2. 用范围for：所有容器 + 只需要**从头到尾遍历** + 不需要索引，**开发首选**；
    
3. 用原生迭代器：需要指定遍历区间、反向遍历、对接STL算法，或者复杂容器(map/set/list)。

## 六、迭代器 避坑指南 & 最佳实践（必记，少踩99%的坑）

### ✅ 坑1：迭代器失效（C++迭代器最大坑，重中之重）

#### 什么是失效？

迭代器本质是指针，当容器**底层内存结构改变/元素位置改变**时，迭代器变成「野指针」，操作会导致程序崩溃/未定义行为。

#### 哪些操作会失效？

1. vector/deque（连续内存）：push_back扩容、insert插入、erase删除 → **全部/部分迭代器失效**；
    
2. list/map/set（链表/红黑树）：仅**指向被删除元素的迭代器**失效，其余均有效；
    

#### 解决办法：

修改容器后，**重新获取迭代器**；erase/insert会返回新的有效迭代器，直接赋值即可：

```C++
vector<int> vec = {1,2,3};
// 错误写法：erase后it失效，++it崩溃
// vec.erase(it); ++it;
// 正确写法：接收返回的有效迭代器
auto it = vec.begin();
it = vec.erase(it);
```

### ✅ 最佳实践1：永远用 `++it` 代替 `it++`

- 前置自增 `++it`：直接移动迭代器，返回新值，**无任何开销**；
    
- 后置自增 `it++`：生成临时副本保存原值，移动后返回副本，**有拷贝开销**；
    
- 无论什么容器，一律写 `++it`，养成习惯，永远没错。
    

### ✅ 最佳实践2：auto的适用与例外（auto万能吗？）

结论：**99.9%的业务开发中，auto完全覆盖所有迭代器场景**，无脑用即可！

仅有的2个需要手写完整迭代器类型的场景（几乎遇不到，了解即可）：

1. 函数的返回值是迭代器时，需要手动声明返回值类型：
    
    ```C++
    vector<int>::iterator find_val(vector<int> &v, int val) {
        return find(v.begin(), v.end(), val);
    }
    ```
    
2. 声明迭代器变量但暂时不赋值时，auto无法推导类型：
    
    ```C++
    vector<int>::iterator it; // 必须手写
    if(flag) it = vec.begin();
    else it = vec.end();
    ```
    

### ✅ 最佳实践3：优先用 `cbegin/cend` 做只读遍历

语法层面杜绝误修改，代码更健壮，尤其是团队协作时，安全性拉满。

1. 迭代器是指针式对象，核心是「通用访问工具」，解耦容器和算法；

4. 三大遍历方式优先级：范围for > auto迭代器 > 下标遍历；
    
5. 常量迭代器(cbegin)是开发首选，reverse_iterator实现倒序；
    
6. map/set/list只能用迭代器遍历，vector可下标/迭代器任选；
    
7. 迭代器失效是最大坑，修改容器后重新获取迭代器即可；
    
8. 永远用++it，不用it++。
