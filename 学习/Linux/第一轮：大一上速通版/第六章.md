### **Part 1: 算术运算 (Arithmetic)**
*PDF Page 25 & 26 - 必考语法*

Shell 默认所有变量都是**字符串**。`a=1+1`，`echo $a` 会输出 `1+1` 而不是 `2`。你需要显式告诉 Shell "我要算数"。

#### **1.1 `expr` 命令 (老式，但教材重点)**
*   **语法**: `expr op1 operator op2`
*   **三大陷阱 (必考纠错)**:
    1.  **空格**: `expr 1+1` (错) -> `expr 1 + 1` (对)。运算符两边必须有空格。
    2.  **乘法转义**: `expr 5 * 5` (错，`*` 是通配符) -> `expr 5 \* 5` (对)。
    3.  **命令替换**: 要想存入变量，必须反引号。 `res=$(expr 1 + 1)`。

#### **1.2 `$((...))` 结构 (新式，推荐实战)**
*   **语法**: `res=$((1+1))`
*   **优点**: 不用空格，不用转义 `*`。
*   **中括号拓展** `$[calculate]` :语法和双括号完全等价 也是推荐用法之一
	* `num=$[1+1];echo $num`
*   **let命令赋值** : `let num=1+1;echo $num` 

> **📝 深度例题 1**
> 编写脚本，计算变量 `a` 和 `b` 的乘积，并存入 `c`。要求用 `expr` 写法。
> **代码**: `c=$(expr $a \* $b)`

### **Part 2: 流程控制 - if 语句 (Conditional Statements)**
*PDF Page 26 & 28 - 核心逻辑*

Shell 的 `if` 语法非常怪异（空格！空格！空格！）。

#### **2.1 语法结构**
```bash
if [ condition ]    # 注意 [ ] 里面的空格
then
    commands
elif [ condition ]
then
    commands
else
    commands
fi                  # 结束符是 if 倒过来
```

#### **2.2 `test` 命令与判断符 (PDF Page 28 表格 - 必须死记)**
Shell 不用 `>` `<` `==` 来比较数字（那是重定向！）。它有一套专门的符号。

| 类型       | 符号                     | 含义                |
| :------- | :--------------------- | :---------------- |
| **数字比较** | **`-eq`**              | Equal (等于)        |
|          | **`-ne`**              | Not Equal (不等于)   |
|          | **`-gt`**              | Greater Than (大于) |
|          | **`-lt`**              | Less Than (小于)    |
|          | **`-ge`** / **`-le`**  | 大于等于 / 小于等于       |
| **文件测试** | **`-f file`**          | 是普通文件吗？           |
|          | **`-d file`**          | 是目录吗？             |
|          | **`-e file`**          | 文件存在吗？(Exist)     |
|          | **`-r` / `-w` / `-x`** | 有读/写/执行权限吗？       |

> **📝 深度例题 2 (高频考题)**
> 题目：检查文件 `data.txt` 是否存在。如果存在，打印 "Found"；否则，创建它。
>
> ``` bash
> filename="data.txt"
> if [ -e "$filename" ]   # 记得加引号防止文件名有空格
> then
>     echo "Found"
> else
>     touch "$filename"
> fi
> ```

### **Part 3: 流程控制 - case 语句 (多路分支)**
*PDF Page 26 - 菜单脚本专用*

如果选项很多，用 `if` 太累，用 `case`。

*   **语法特征**:
    *   以 `case $var in` 开始。
    *   每个模式以 `)` 结束。
    *   每个执行块以 **`;;` (双分号)** 结束。
    *   默认分支用 **`*)`** 表示。
    *   以 **`esac`** (case 倒过来) 结束。

> **📝 深度例题 3 (结构填空)**
> 补全代码：用户输入 "start" 则启动，"stop" 则停止。
> ```bash
> read cmd
> case $cmd in
>     "start")
>         echo "Starting..."
>         ;;      # 填空点 1
>     "stop")
>         echo "Stopping..."
>         ;;
>     *)          # 填空点 2：默认分支
>         echo "Unknown"
>         ;;
> esac
> ```

### **Part 4: 循环结构 (Loops)**
*PDF Page 27 & 28*

#### **4.1 `for` 循环 (遍历列表)**
`for i in {a..b}` 
```bash
for var in 1 2 3 4 5
do
    echo "Number $var"
done
```
*   **实战技巧**: `for file in *.txt` (遍历所有文本文件)。

#### **4.2 `while` 循环 (条件为真时执行)**
```bash
count=1
while [ $count -le 5 ]
do
    echo $count
    count=$((count + 1))  # 记得自增，否则死循环
done
```

#### **4.3 循环控制 (Break & Continue)**
*PDF Page 29*
*   **`break`**: 跳出整个循环。
*   **`continue`**: 跳过本次迭代，进入下一次。

### **Part 5: 命令行参数 (Positional Parameters)**
*PDF Page 30 - 脚本交互核心*

当你在终端运行 `./script.sh apple banana` 时，脚本内部如何获取这两个参数？

*   **`$0`**: 脚本名 (`./script.sh`)
*   **`$1`**: 第 1 个参数 (`apple`)
*   **`$2`**: 第 2 个参数 (`banana`)
*   **`$#`**: 参数总个数 (`2`) —— *常用于 `if [ $# -lt 1 ]` 判断用户有没有输入参数。*
*   **`$@`**: 所有参数的列表 (`apple banana`) —— *常用于 `for i in $@` 遍历所有参数。*

#### **`shift` 命令 (位移)**
*PDF Page 30*
*   `shift` 会把 `$2` 变成 `$1`，`$3` 变成 `$2`，原来的 `$1` 被丢弃。
*   **作用**: 用来轮询处理参数。

### **Chapter 6 终极编程挑战**



**题目：编写一个脚本 `backup_check.sh`**
1.  该脚本接收**一个文件名**作为参数。
2.  **检查参数个数**：如果没有提供参数，输出 "Error: No file specified" 并退出。
3.  **检查文件存在性**：如果该文件**不**存在，输出 "File not found"。
4.  **执行备份**：如果文件存在，判断它是否**可读 (`-r`)**。
    *   如果可读，把它复制为 `filename.bak` 并输出 "Backed up"。
    *   如果不读，输出 "Permission denied"。

**(请在输入框写出代码，不需要太完美，逻辑对就行。)**


```bash
#!/bin/bash
file="test.txt"
if [ ! -f $file ]
then 
echo "no"
fi
if [ -r $file ]
then
cp $file ${file}.bak
echo "backed up"
else
echo  "permission denied"
fi


```
