### **Part 1: Shell 的启动机制与执行原理 (Execution Mechanisms)**
*PDF Page 19 & 22 - 深度解析*

当你在终端敲下 `./script.sh` 时，操作系统底层到底发生了什么？这涉及 **Process (进程)** 的概念。

#### **1.1 父 Shell 与 子 Shell (Parent vs Child Shell)**
*   **Login Shell**: 你登录系统时获取的那个 Shell（比如 bash），它是**父进程**。
*   **Execution**:
    *   当你运行脚本时，父 Shell 会调用系统函数 `fork()` 复制出一个自己，叫做 **子 Shell**。
    *   脚本里的代码是在这个**子 Shell** 里跑的。
    *   **跑完销毁**：子 Shell 里的变量（局部变量）随之消失，不会影响父 Shell。

#### **1.2 三种执行方式的底层差异 (核心考点)**
*PDF Page 22*

| 命令 | 是否产生子 Shell? | 权限要求 | 影响当前环境? | 应用场景 |
| :--- | :--- | :--- | :--- | :--- |
| **`./script.sh`** | **Yes** (Fork) | 需要 `x` | No | 运行普通工具 |
| **`bash script.sh`** | **Yes** (Fork) | 只要 `r` | No | 调试脚本 |
| **`source script.sh`** | **No** (当前进程) | 只要 `r` | **Yes** | **加载配置文件** |

> **🧪 深度例题 1 (Trap Question)**
> 假设脚本 `set_path.sh` 内容为：`export PATH=$PATH:/my/new/dir`
>
> 1.  如果用 `./set_path.sh` 运行，运行完后，你在终端敲 `echo $PATH`，能看到新路径吗？
>     *   **答案**: **不能**。因为子 Shell 修改了 PATH，子 Shell 死后，父 Shell 的 PATH 没变。
> 2.  如果用 `source set_path.sh` 运行呢？
>     *   **答案**: **能**。因为是在当前 Shell 直接执行。

### **Part 2: 变量的作用域与环境 (Variable Scope)**
*PDF Page 23 - 局部 vs 全局*

#### **2.1 Local vs Global (Export)**
*   **Local (局部)**: 默认定义的变量（`name="Steve"`）只在当前 Shell 有效。子 Shell 看不到。
*   **Global (全局)**: 使用 **`export`** 命令，可以让变量“遗传”给子 Shell。

**原理图解：**
`Father Shell (name="A")` -> Fork -> `Child Shell (name is Empty)`
`Father Shell (export name="A")` -> Fork -> `Child Shell (name="A")`

> **📝 深度编程题 2**
> 编写两个脚本验证 export。
> *   `parent.sh`: 定义变量 `city="NY"`，然后调用 `./child.sh`。
> *   `child.sh`: 输出 `echo "City is $city"`。
>
> **问**: 如果不加 `export`，运行 `parent.sh` 会输出什么？
> **答**: `City is ` (空的)。
> **问**: 如何修改 `parent.sh` 让它输出 `City is NY`？
> **答**: `export city="NY"`

#### **2.2 `read` 命令的参数陷阱**
*PDF Page 23*
`read` 不仅仅是读取，它能处理格式。虽然 PDF 没细写参数，但考试常考：
*   **`read -p "Prompt" var`**: 在同一行显示提示语。
    *   *对比*: `echo "Enter name"; read name` vs `read -p "Enter name: " name`。后者体验更好。

### **Part 3: 别名与历史 (Aliases & History)**
*PDF Page 20*

这是提高效率的神器。

#### **3.1 Command Alias (别名)**
*   **定义**: `alias name='command'`
*   **例子**: `alias ll='ls -l'`
*   **持久化**: 如果你只在终端敲这行命令，重启就没了。**必须写入 `~/.bashrc` 文件** (PDF 虽未明说，但这是 Linux 常识，若是大题问如何永久生效，必答)。

#### **3.2 History**
*   **`history`**: 显示你敲过的所有命令。
*   **`!n`**: 执行历史记录里第 n 条命令。
*   **`!!`**: 执行**上一条**命令 (sudo !! 很有用)。

### **Part 4: 前后台作业控制 (Job Control)**
*PDF Page 20 - 容易混淆的操作*

Linux 是多任务的，你可以把脚本扔到后台跑。

*   **Foreground (前台)**: 霸占你的终端，你不能干别的。
*   **Background (后台)**: 在命令后加 **`&`**。
    *   `./long_script.sh &`
*   **挂起 (Suspend)**: **`Ctrl + Z`**。*注意：这没结束进程，只是暂停了并扔到后台。*
*   **管理命令**:
    *   **`jobs`**: 查看后台有哪些任务。
    *   **`fg %1`**: 把后台第 1 个任务拉回前台 (Foreground)。
    *   **`bg %1`**: 让后台暂停的任务继续运行 (Background)。

> **📝 深度例题 3 (场景模拟)**
> 你运行了一个脚本 `./backup.sh`，发现它太慢了，卡住了终端。你想让它在后台继续跑，把终端还给你。请写出操作步骤。
>
> **标准操作**:
> 1.  按 **`Ctrl + Z`** (暂停并在终端显示 `[1]+ Stopped`)。
> 2.  输入 **`bg`** (让它在后台继续跑)。
> 3.  输入 **`jobs`** 确认状态 (应显示 `Running`)。
