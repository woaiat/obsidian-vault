# AI 导师学习流目标

这部分才是以后真正的 2-sigma 学习代理核心。

## 理想目标

当你说：

`我要学高数第九章，这里是教材和材料`

系统应该能做到：

1. 完整读完你指定的材料
2. 识别本章定义、定理、例题、题型和前置依赖
3. 按教材定义顺序组织教学
4. 对新知识采用老师带课模式
5. 对复习内容采用苏格拉底问答模式
6. 每个点都有过线判断
7. 自动记录学到哪里、哪里没过线、下次从哪继续

## 目标形态

这条学习流最终应该是：

`材料输入 -> 完整预读 -> 正式带课 -> 中途检测 -> 不过线回退 -> 会话结束沉淀 -> 间隔复习`

## 为什么这才是核心

因为这一步如果不成立，前面的模板、后面的 App 都只是外壳。

## 这层和底座的关系

- 底座负责统一规则
- 教学引擎负责把规则变成真正的教学过程
- 执行模板负责在当前阶段先承接这个过程
