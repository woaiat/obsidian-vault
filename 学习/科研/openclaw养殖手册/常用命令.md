| **动作**           | **指令**                                                    | **说明**                      |
| ---------------- | --------------------------------------------------------- | --------------------------- |
| **一键点火**         | `docker-compose up -d`                                    | 后台启动所有服务。                   |
| **热重启**          | `docker-compose restart`                                  | 不销毁容器，只重启服务（改了 `.env` 后常用）。 |
| **熄火撤收**         | `docker-compose down`                                     | 停止并移除容器（释放内存空间）。            |
| **强制重构**         | `docker-compose up -d --build`                            | 当你大幅修改代码或配置文件后，确保镜像更新。      |
| **动作**           | **指令**                                                    | **说明**                      |
| **进入 TUI**       | `docker-compose exec openclaw-gateway openclaw tui`       | **最常用**。进入终端交互界面。           |
| **获取 Dashboard** | `docker-compose exec openclaw-gateway openclaw dashboard` | 获取带 Token 的 Web 端登录链接。      |
| **重置配置**         | `docker-compose exec openclaw-gateway openclaw onboard`   | 重新触发 API Key 或性格设置的引导。      |
|                  |                                                           |                             |
|                  |                                                           |                             |

|**动作**|**指令**|**说明**|
|---|---|---|
|**实时日志**|`docker-compose logs -f --tail 50`|实时查看后台发生了什么。|
|**容器审计**|`docker ps -a`|查看容器状态（是否在 Starting 或 Healthy）。|
|**空间清理**|`docker system prune`|**慎用**。清理所有未使用的镜像和缓存。|
