## 2.迭代与递归
### 2.1. 迭代
**for    while    嵌套循环** 
### 2.2.递归
- 终止条件：什么时候由递转归
- 递归调用：“递”
- 返回结果：“归”  结果返回上一级

``` C++
/* 递归 */
int recur(int n) {
    // 终止条件
    if (n == 1)
        return 1;
    // 递：递归调用
    int res = recur(n - 1);
    // 归：返回结果
    return n + res;
}
```

**对比迭代和递归：**
- 迭代：自下而上的解决问题 从最基础的步骤开始 重复累加 直到任务完成
- 递归：自上而下的解决问题 将原问题分解为更小的子问题 直到分解为最基本的情况（解已知）停止

#### 1.调用栈
**递归函数每次调用自身时 系统为其开辟新的函数分配内存** 故过深的递归容易导致栈溢出
- 递归通常比迭代更加耗费内存空间
- 递归通常比循环的时间效率更低

#### 2.尾递归
``` C++
/* 尾递归 */
int tailRecur(int n, int res) {
    // 终止条件
    if (n == 0)
        return res;
    // 尾递归调用
    return tailRecur(n - 1, res + n);
}
```

**如果在函数返回前的最后一步才进行递归调用 则空间效率上与迭代相当** 
- **普通递归：** 每一步操作在“归”中执行 每层返回后都要再执行一次求和操作 在函数过程中递归
- **尾递归：** 每一步操作在“递”中执行 “归”的过程只需要层层返回 在函数最后一步递归


#### 3.递归树
处理“分治”相关算法问题时 递归往往比迭代更易读 更直观  递归天然更加适合处理树 图 分治 链表 相关问题

```c++
/* 斐波那契数列：递归 */
int fib(int n) {
    // 终止条件 f(1) = 0, f(2) = 1
    if (n == 1 || n == 2)
        return n - 1;
    // 递归调用 f(n) = f(n-1) + f(n-2)
    int res = fib(n - 1) + fib(n - 2);
    // 返回结果 f(n)
    return res;
}
```


### 2.3. 递归和迭代的对比
递归：**其工作机制与栈的先入后出有异曲同工之妙**
- 递：在“调用栈”上分配新的栈帧
- 归：移除栈帧

**模拟将递归转化为迭代：**  
```c++
/* 使用迭代模拟递归 */
int forLoopRecur(int n) {
    // 使用一个显式的栈来模拟系统调用栈
    stack<int> stack;
    int res = 0;
    // 递：递归调用
    for (int i = n; i > 0; i--) {
        // 通过“入栈操作”模拟“递”
        stack.push(i);
    }
    // 归：返回结果
    while (!stack.empty()) {
        // 通过“出栈操作”模拟“归”
        res += stack.top();
        stack.pop();
    }
    // res = 1+2+3+...+n
    return res;
}
```
