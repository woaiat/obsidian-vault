## 2.迭代与递归
### 2.1. 迭代
**for    while    嵌套循环** 
### 2.2.递归
- 终止条件：什么时候由递转归
- 递归调用：“递”
- 返回结果：“归”  结果返回上一级

``` C++
/* 递归 */
int recur(int n) {
    // 终止条件
    if (n == 1)
        return 1;
    // 递：递归调用
    int res = recur(n - 1);
    // 归：返回结果
    return n + res;
}
```

**对比迭代和递归：**
- 迭代：自下而上的解决问题 从最基础的步骤开始 重复累加 直到任务完成
- 递归：自上而下的解决问题 将原问题分解为更小的子问题 直到分解为最基本的情况（解已知）停止

#### 1.调用栈
**递归函数每次调用自身时 系统为其开辟新的函数分配内存** 故过深的递归容易导致栈溢出
- 递归通常比迭代更加耗费内存空间
- 递归通常比循环的时间效率更低

#### 2.尾递归
``` C++
/* 尾递归 */
int tailRecur(int n, int res) {
    // 终止条件
    if (n == 0)
        return res;
    // 尾递归调用
    return tailRecur(n - 1, res + n);
}
```

**如果在函数返回前的最后一步才进行递归调用 则空间效率上与迭代相当** 
- **普通递归：** 每一步操作在“归”中执行 每层返回后都要再执行一次求和操作 在函数过程中递归
- **尾递归：** 每一步操作在“递”中执行 “归”的过程只需要层层返回 在函数最后一步递归


#### 3.递归树
处理“分治”相关算法问题时 递归往往比迭代更易读 更直观  递归天然更加适合处理树 图 分治 链表 相关问题

```c++
/* 斐波那契数列：递归 */
int fib(int n) {
    // 终止条件 f(1) = 0, f(2) = 1
    if (n == 1 || n == 2)
        return n - 1;
    // 递归调用 f(n) = f(n-1) + f(n-2)
    int res = fib(n - 1) + fib(n - 2);
    // 返回结果 f(n)
    return res;
}
```


### 2.3. 递归和迭代的对比
递归：**其工作机制与栈的先入后出有异曲同工之妙**
- 递：在“调用栈”上分配新的栈帧
- 归：移除栈帧

**模拟将递归转化为迭代：**  
```c++
/* 使用迭代模拟递归 */
int forLoopRecur(int n) {
    // 使用一个显式的栈来模拟系统调用栈
    stack<int> stack;
    int res = 0;
    // 递：递归调用
    for (int i = n; i > 0; i--) {
        // 通过“入栈操作”模拟“递”
        stack.push(i);
    }
    // 归：返回结果
    while (!stack.empty()) {
        // 通过“出栈操作”模拟“归”
        res += stack.top();
        stack.pop();
    }
    // res = 1+2+3+...+n
    return res;
}
```



## 3.时间复杂度
**时间复杂度统计的是算法运行时间随着数据量变大时的增长趋势**
### 3.1 统计时间增长趋势
```c++
// 算法 A 的时间复杂度：常数阶
void algorithm_A(int n) {
    cout << 0 << endl;
}
// 算法 B 的时间复杂度：线性阶
void algorithm_B(int n) {
    for (int i = 0; i < n; i++) {
        cout << 0 << endl;
    }
}
// 算法 C 的时间复杂度：常数阶
void algorithm_C(int n) {
    for (int i = 0; i < 1000000; i++) {
        cout << 0 << endl;
    }
}
```

- 算法 `A` 只有 1 个打印操作，算法运行时间不随着  增大而增长。我们称此算法的时间复杂度为“常数阶”。
- 算法 `B` 中的打印操作需要循环 n 次，算法运行时间随着 n 增大呈线性增长。此算法的时间复杂度被称为“线性阶”。
- 算法 `C` 中的打印操作需要循环 1000000 次，虽然运行时间很长，但它与输入数据大小 n 无关。因此 `C` 的时间复杂度和 `A` 相同，仍为“常数阶”。


### 3.2 函数渐进上界

```c++
void algorithm(int n) {
    int a = 1;  // +1
    a = a + 1;  // +1
    a = a * 2;  // +1
    // 循环 n 次
    for (int i = 0; i < n; i++) { // +1（每轮都执行 i ++）
        cout << 0 << endl;    // +1
    }
}
```

设算法的操作数量是一个关于输入数据大小$n$的函数，记为 $T(n)$ ，则以上函数的操作数量为：

			$T(n) = 3 + 2n$ 

 $T(n)$ 是一次函数，说明其运行时间的增长趋势是线性的，因此它的时间复杂度是线性阶。

我们将线性阶的时间复杂度记为 $O(n)$  ， 这个数学符号称为大 $O$  记号（big- notation），表示函数 $T(n)$  的渐近上界（asymptotic upper bound）。

时间复杂度分析本质上是计算“操作数量 $T(n)$ ”的渐近上界，它具有明确的数学定义。

### 3.3. 推算方法
**第一步：统计操作数量
  
**$T(n)$ 中的各种系数、常数项都可以忽略**。

1. **忽略 $T(n)$ 中的常数**。
2. **省略所有系数**。例如，循环 $2n$ 次、$5n + 1$ 次等，都可以简化记为 $n$ 次
3. **循环嵌套时使用乘法**。总操作数量等于外层循环和内层循环操作数量之积，每一层循环依然可以分别套用第 `1.` 点和第 `2.` 点的技巧。

**第二步：判断渐近上界

**时间复杂度由 $T(n)$ 中最高阶的项来决定**。
